# Skills Registry

## 目的
防止"装了就忘"，记录每个已安装skill的基础信息。

## 规则
- 只记录基础信息，不做智能匹配
- 一句话功能描述
- 明确适用和不适用场景
- 记录风险级别

---

## 已安装Skills

### 1. skill-vetter
- **功能：** 安全审查员，在安装其他skills前检查安全风险
- **适用场景：** 安装新skill前的安全审查
- **不适用场景：** 已安装skill的日常使用
- **风险级别：** 低
- **调用成本：** 低（只需读取skill代码）
- **最近使用：** 2026-03-08（审查using-superpowers）

---

### 2. self-improving-agent
- **功能：** 自我进化引擎，跨会话持续学习，记录错误和成功经验
- **适用场景：** 需要记录学习经验、避免重复错误
- **不适用场景：** 一次性任务、不需要长期记忆的场景
- **风险级别：** 低
- **调用成本：** 低（自动运行）
- **最近使用：** 未明确使用（可能在后台运行）

---

### 3. using-superpowers
- **功能：** 行为纪律框架，强制"先查skill再行动"
- **适用场景：** 每次任务开始前，检查是否有相关skill
- **不适用场景：** 紧急任务、明确不需要skill的简单操作
- **风险级别：** 低
- **调用成本：** 低（只是行为规范）
- **最近使用：** 应该每次任务都使用（但虾哥还没养成习惯）

### 4. verification-before-completion

- **适用场景：** 任何需要报告"完成"的任务
- **不适用场景：** 中间过程、不需要验证的简单操作
- **风险级别：** 低
- **调用成本：** 低（验证步骤）
- **最近使用：** 应该每次报告完成前使用（但虾哥还没养成习惯）
