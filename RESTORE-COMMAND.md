## 🚨 错误处理

### 错误 1：文件不存在

**如果某个核心文件不存在：**
1. 记录缺失的文件名
2. 继续读取其他文件
3. 在汇报时说明："⚠️ 缺失文件：[文件名]"
4. 提供修复建议："请检查 Git 仓库是否完整"

---

### 错误 2：文件读取失败

**如果某个文件读取失败（权限、编码等问题）：**
1. 记录失败的文件名和错误信息
2. 继续读取其他文件
3. 在汇报时说明："⚠️ 读取失败：[文件名] - [错误信息]"

---

### 错误 3：记忆文件为空

**如果 memory/ 目录为空或没有最近文件：**
1. 说明这是全新部署或记忆丢失
2. 在汇报时说明："📅 记忆范围：无历史记忆（全新部署或记忆丢失）"
3. 建议："如果这不是全新部署，请检查 Git 仓库"

---

### 错误 4：验证失败

**如果自检时发现问题：**
1. 记录未通过的验证项
2. 在汇报时说明："⚠️ 验证未通过：[验证项]"
3. 提供修复建议
