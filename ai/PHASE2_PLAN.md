# Phase 2: 规则引擎 V1 - 实施计划

## 🎯 目标

用简单规则过滤交易信号，提升胜率

**验收标准：**
- 过滤准确率 > 60%
- 误删率 < 20%

---

## 📋 三大任务

### 任务 1：粗筛规则（第 1 周）

**目的：** 快速排除明显不适合交易的市场环境

**规则清单：**

1. **趋势过滤**
   - MA20 > MA50 → LONG_ONLY
   - MA20 < MA50 → SHORT_ONLY
   - 趋势不明确 → BOTH

2. **波动率过滤**
   - ATR < 平均值 * 0.5 → NONE
   - ATR > 平均值 * 2.0 → NONE

3. **成交量过滤**
   - Volume < 平均值 * 0.7 → NONE

4. **时间过滤**
   - 重大新闻时段 → NONE
   - 周末/节假日 → NONE

**文件：** `ai/coarse_filter.py`

---

### 任务 2：精筛规则（第 2 周）

**目的：** 在粗筛基础上细致优化

**规则清单：**

1. **RSI 超买超卖**
   - RSI > 70 → SHORT_ONLY
   - RSI < 30 → LONG_ONLY

2. **箱体质量**
   - 箱体评分 < 80 → NONE

3. **价格位置**
   - 接近上边界 → SHORT_ONLY
   - 接近下边界 → LONG_ONLY
   - 中间位置 → BOTH

**文件：** `ai/fine_filter.py`

---

### 任务 3：离线回测框架（第 3 周）

**目的：** 验证规则效果，调整参数

**步骤：**

1. **准备历史数据**
   - 采集 3 个月历史 K 线
   - 保存到 SQLite

2. **模拟信号生成**
   - 用历史数据模拟箱体信号

3. **应用规则过滤**
   - 对每个信号应用粗筛 + 精筛

4. **统计分析**
   - 胜率、误删率、保留率
   - 生成回测报告

**文件：** `ai/backtest.py`

## ✅ 验收标准

**Phase 2 完成标准：**

1. **过滤准确率 > 60%**
   - 保留的信号中，盈利信号占比 > 60%

2. **误删率 < 20%**
   - 删掉的信号中，盈利信号占比 < 20%

3. **回测报告完整**
   - 包含：胜率、误删率、保留率、平均盈亏
   - 包含：参数敏感性分析
   - 包含：优化建议

4. **代码质量**
   - 所有函数有单元测试
   - 代码有注释
   - 参数可配置

---

## 💡 优化策略

**如果误删率高（> 20%）：**
- 放宽 ATR 阈值
- 放宽成交量阈值
- 减少精筛规则

**如果胜率低（< 60%）：**
- 收紧 ATR 阈值
- 增加精筛规则
- 调整 RSI 阈值

**如果保留率过低（< 30%）：**
- 规则过于严格
- 需要放宽部分条件
