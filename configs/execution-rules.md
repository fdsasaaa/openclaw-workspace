# 当前执行保护规范

**生效日期**: 2026-03-05  
**适用范围**: 所有 OpenClaw 任务，包括恢复阶段、长耗时操作、自动化脚本  
**版本**: v1.0

---

## 规则1：任务拆分与小步验证

**规范**: 长任务必须拆分为 ≤10 步的子任务，每步执行后需验证结果，确认通过后再进入下一步。

**执行要求**:
- 任务开始前先输出拆分计划
- 每步完成后输出结果摘要
- 验证通过后方可进入下一步
- 验证失败时回退或修正后再继续

---

## 规则2：检查点保存与续跑

**规范**: 每完成一步必须输出阶段结果摘要，并写入检查点文件；掉线后优先读取检查点续跑，禁止整段重做。

**检查点文件格式**:
```json
{
  "taskId": "任务标识",
  "createdAt": "2026-03-05T14:00:00",
  "updatedAt": "2026-03-05T14:30:00",
  "steps": [
    {"step": 1, "status": "done", "result": "..."},
    {"step": 2, "status": "in_progress", "checkpoint": "..."},
    {"step": 3, "status": "pending"}
  ],
  "lastModel": "kimi-coding/k2p5",
  "contextTokens": 45000
}
```

**落点目录**: `C:\OpenClaw_Workspace\memory\progress-<taskid>.json`

**续跑触发**: 启动时检查 `memory\progress-*.json`，如有 `in_progress` 状态，询问是否续跑。

---

## 规则3：失败处理与退避

**规范**: 遇到失败时，严格按照以下顺序处理，禁止自动无限重试。

**处理顺序**:
1. **同模型退避重试 1 次**: 等待 5 秒后重试
2. **fallback 尝试 1 次**: 等待 15 秒后切换至 fallback 模型
3. **停止并汇报**: 仍失败则立即停止，向用户汇报错误详情

**可重试错误**（执行退避）:
- 429 Too Many Requests
- 503 Service Unavailable
- 524/504 Gateway Timeout
- 网络超时/连接重置
- 模型过载响应慢

**不可重试错误**（立即停止）:
- 401 Unauthorized（密钥错误）
- 403 Forbidden（权限不足）
- 400 Bad Request（参数错误）
- 内容审核拒绝
- 上下文长度超限（需人工分段）

---

## 规则4：Token 与上下文控制

**规范**: 主动监控和控制上下文膨胀，避免 token 消耗过快。

**预警线**: 单 session 达到 150k tokens 时主动提醒

**硬上限**: 超过 200k tokens 必须强制分段（写入文件，新开 session）

**清理策略**: 每 10 轮对话主动提议 `/compact` 或归档旧消息

**成本追踪**: 在 `memory\usage-YYYY-MM-DD.json` 记录每日 token 消耗

---

## 规则5：双区并行输出规范

**规范**: 严格按照双区并行策略输出文件。

**运行区** (`C:\Users\ME\.openclaw\workspace\`):
- OpenClaw 核心配置（AGENTS.md, SOUL.md, USER.md, IDENTITY.md, TOOLS.md, HEARTBEAT.md）
- 会话状态（系统自动管理）
- 身份配置（只读保护）

**项目主目录** (`C:\OpenClaw_Workspace\`):
- 数据文件 (`Data\`)
- 脚本文件 (`step02\`, 根目录 `.ps1`)
- 报告输出 (`reports\`)
- 检查点 (`memory\`)
- 项目代码 (`projects\`)
- 用户日志 (`logs\`)
- 临时文件 (`temp\`)
- 配置规则 (`configs\`)

---

## 暂不自动化（需人工确认）

以下功能当前阶段仍需人工确认，不自动执行：

- 检查点文件的自动写入（当前需人工确认后写入）
- 失败类型的自动判断（当前需人工确认是否可重试）
- Token 用量的实时监控与自动分段（当前仅提醒，不强制）
- 上下文压缩 `/compact` 的自动触发（当前需人工决策）
