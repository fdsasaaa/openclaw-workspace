# OpenClaw Hook 配置脚本

## 📋 说明

OpenClaw 的内置 Hook 机制主要用于系统级功能（如 boot-md、session-memory）。
自定义 Hook（如错误日志、用户纠正）需要通过其他方式实现。

---

## 🎯 实施方案：手动 Hook（在 AGENTS.md 中定义规则）

由于 OpenClaw 的 Hook 配置格式较为复杂，我建议采用**手动 Hook**方案：

### 方案：在 AGENTS.md 中添加自动化规则

我会在每次交互时主动执行以下规则：

#### 1. 错误检测规则
```markdown
## 错误自动记录规则

每当工具调用失败时，我会：
1. 立即记录到 memory/errors.md
2. 分析失败原因
3. 提供替代方案
4. 避免重复尝试（同一错误 3 次内不再重试）
```

#### 2. 用户纠正规则
```markdown
## 用户纠正自动记录规则

当用户消息包含以下关键词时，我会自动记录：
- "不对"、"错了"、"实际上"、"应该是"
- "其实"、"Actually"

记录内容：
- 我的错误回答
- 用户的纠正
- 正确答案
- 经验教训
```

#### 3. 需求捕获规则
```markdown
## 需求自动记录规则

当用户提出新需求时，我会自动记录到 memory/requirements.md：
- "能不能"、"可以帮我"、"希望你"
- "以后"、"下次"

记录内容：
- 需求描述
- 优先级（高/中/低）
- 实施难度
- 预计时间
```
