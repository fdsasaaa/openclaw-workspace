# 任务路由系统 (Task Router)

## 任务分类标准

### 🟢 由我（虾哥）处理

**适合场景：**
- 快速问答（< 5分钟）
- 信息搜索、总结
- 单文件简单编辑
- 状态检查、文件浏览
- 配置查询、解释说明

**示例：**
- "这个文件里有什么？"
- "搜索一下 xxx 的资料"
- "帮我改一下这行配置"
- "系统状态怎么样？"

---

### 🔵 由 Claude Code 处理

**适合场景：**
- 多文件代码修改
- 复杂重构（> 3个文件）
- 新项目初始化
- 调试复杂问题
- 代码审查、优化建议
- 需要 IDE 功能的任务

**示例：**
- "重构整个项目的错误处理"
- "帮我搭建一个新项目的脚手架"
- "审查这个 PR 的代码"
- "优化这个函数的性能"

---

### 🟡 协作模式

**适合场景：**
- 复杂需求需要先分析拆解
- 需要我收集信息后再编码
- Claude Code 完成后需要我总结

**流程：**
1. 我分析需求 → 拆解任务
2. 启动 Claude Code 执行
3. Claude Code 工作 → 可能需要确认
4. 我监控进度 → 必要时介入
5. 完成后我总结结果

---

## 决策流程图

```
用户提出任务
    ↓
任务复杂度评估
    ↓
┌─────────────┬─────────────┬─────────────┐
│  简单快速    │   复杂编码   │   需要分析   │
│  (<5分钟)   │  (多文件)   │   + 执行    │
└──────┬──────┴──────┬──────┴──────┬──────┘
       ↓             ↓             ↓
   我直接处理    启动 Claude    协作模式
       ↓             ↓             ↓
   即时响应      监控执行       我 → Claude → 我
```

---

## 与 Claude Code 的协作协议

### 启动时
- 明确任务目标
- 提供必要的上下文
- 设定工作范围

### 执行中
- 监控 Claude Code 的输出
- 处理需要确认的请求（Yes/No）
- 必要时提供额外信息

### 完成后
- 检查结果完整性
- 验证关键修改
- 总结变更内容
- 报告给用户

## 质量检查清单

### 代码任务检查
- [ ] 文件是否正确修改
- [ ] 语法是否正确
- [ ] 功能是否完整
- [ ] 是否有遗漏的文件
- [ ] 测试是否通过（如果有）

### 项目任务检查
- [ ] 目录结构是否正确
- [ ] 配置文件是否完整
- [ ] 依赖是否正确安装
- [ ] 是否能正常运行
