### 项目背景

### 关键问题和解决

#### 问题 1：编译错误（13 个错误）
**错误类型：**
1. 函数重复定义（ExtractBoxIdFromComment）
2. 函数名错误（StringSubstring → StringSubstr）
3. OrderDelete 不存在（需要用 OrderSend + TRADE_ACTION_REMOVE）

**修复：**
- 删除重复函数定义
- 修正 MQL5 函数名
- 使用标准 MQL5 订单删除方法

**结果：** 0 errors, 0 warnings ✅

---

#### 问题 2：JSON 解析失败（编码问题）
**现象：**
- EA 能打开文件，但读取到 0 行
- 文件实际有内容（257 字节）

**根本原因：**
- Python 使用 UTF-8 编码写入
- EA 使用 FILE_ANSI 标志读取
- 编码不匹配导致读取失败

**解决方案：**
1. Python 改用 ANSI (cp1252) 编码写入
2. EA 使用 FILE_ANSI 标志读取
3. Webhook 接收器同步改为 ANSI 编码

**修改文件：**
- `test_single_event.py` - 测试脚本
- `webhook_receiver.py` - Webhook 接收器
- `Tradingview策略AI配合EA.mq5` - EA 文件读取

**结果：** EA 成功读取并解析 JSON ✅

---

#### 问题 3：JSON 解析逻辑错误
**现象：**
- EA 读取到内容，但 event 字段为空
- 日志显示"未知事件类型: []"

**根本原因：**
- 原始解析函数只匹配 `"event":"BOX_NEW"`（无空格）
- 实际 JSON 格式为 `"event": "BOX_NEW"`（有空格）
- 硬编码搜索字符串导致匹配失败

**解决方案：**
修复 4 个 JSON 解析函数：
1. `ParseEventType()` - 事件类型解析
2. `ParseJSONString()` - 字符串字段解析
3. `ParseJSONInt()` - 整数字段解析
4. `ParseJSONDouble()` - 浮点数字段解析

**修复策略：**
- 只搜索字段名（如 `"event"`），不包含冒号
- 手动跳过冒号和空格
- 找到开始引号后提取值

**结果：** 成功解析所有字段 ✅

### 架构讨论：方案 A vs 方案 B
