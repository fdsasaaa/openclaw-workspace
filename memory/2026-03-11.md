## 08:34 - Cron 和 Heartbeat 机制优化

✅ **学习了 OpenClaw 进阶教程（Cron + Heartbeat）**

**关键收获：**
1. Main vs Isolated 模式的本质区别
2. Main 模式的坑：依赖心跳轮询，可能延迟或不执行
3. Isolated 模式的优势：独立会话，到点就干，干完主动汇报

**执行的改进：**
1. ✅ 将"每日 Git 备份"任务从 Main 模式升级为 Isolated 模式
   - 原因：确保凌晨2点准时执行，不依赖心跳
   - 添加了超时保护（120秒）和主动汇报机制
2. ✅ 设计了 HEARTBEAT.md 巡检清单
   - 周一三五：磁盘/网关/Git备份
   - 周二四六：系统更新/文件整理/内存归档
   - 每天：通知检查/日志创建
   - 静默规则：正常返回 HEARTBEAT_OK，有问题才汇报
3. ✅ 更新 AGENTS.md，添加"Cron 最佳实践"章节
   - Main vs Isolated 选择表
   - 进阶技巧（合并任务、显式时区、成本优化、错误防护）
   - Cron 表达式格式和调度类型说明

**经验教训：**
- 复杂任务、耗时操作、要求准点执行 → 必须用 Isolated 模式
- 所有 Cron 任务必须显式指定时区（`tz: "Asia/Shanghai"`）
- 心跳适合批量检查，Cron 适合精准调度
