### 教训2：不要过度设计
- **问题：** 收到很多"最佳实践"建议
- **原因：** 很多方案是针对生产环境，不适合当前阶段
- **解决：** 评估每个方案的实际价值，拒绝过度设计

### 核心理念

**Skills 是手，不是工具箱**
- ❌ 不要想："我有哪些 skills？"
- ✅ 要想："遇到这个任务，我的手会自然做什么？"

**习惯养成3个阶段：**
1. 第1周：主动提醒（每次遇到任务，主动检查 skills 清单）
2. 第2-3周：条件反射（看到触发词 → 立即想到 Skill）
3. 第4周+：自然反应（不需要思考，直接就用）

### 成功标准（1个月后）

1. ✅ 所有已安装的 skills 都至少使用过1次
2. ✅ 常用 skills 形成条件反射（<1秒想起来）
3. ✅ 每周使用 skills 的次数 > 10次
4. ✅ "该用但没用"的情况 < 10%
