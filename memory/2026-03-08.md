### 🔍 Qwen替代方案研究
虾哥整理了7个可用的替代方案：
1. **OpenRouter**（最推荐）- 聚合多个模型，有免费额度
2. **字节跳动Doubao（豆包）** - 国内服务，价格便宜，输出标准JSON
3. **GPT-4o / Claude 3.5** - 最稳定，需要付费
4. Google Gemini 2.0 Flash（15天限制）
5. 阿里云通义千问VL（可能有中文引号问题）
6. 百度文心一言4.0（需要测试）
7. 其他OpenAI兼容的API服务

### 💡 收到的AI建议（来自其他大模型）

**核心解决方案：**
1. **Prompt强制约束**（零代码）- 在系统提示中强制要求标准JSON格式
2. **PowerShell预处理**（100%根治）- 在解析前替换中文引号
3. **换成豆包模型**（推荐）- 国产、稳定、输出标准JSON
4. **AutoHotKey + 视觉分析**（不依赖Midscene）- ChatGPT分析K线 → AHK执行画箱体
5. **MQL4脚本**（最精准）- MT4原生支持，一行代码画箱体

**FFmpeg修复方案：**
- 使用 `-movflags +faststart` 参数
- 正确的停止方式（不用Stop-Process -Force）
- 或使用OBS/Windows自带录屏工具

### 🎯 核心发现

**Midscene的局限性：**
- Midscene是封装好的npm包，无法直接修改内部代码
- 无法在Midscene层面添加字符串预处理
- 需要从源头解决（换模型）或绕过（用其他方案）

**可行的完整方案：**
```
截图MT4 → ChatGPT分析K线 → 返回箱体坐标 → AutoHotKey执行画箱体
```
