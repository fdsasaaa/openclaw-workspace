# 2026-03-06 经验记录

## 教训：网页内容分析必须实时获取

**场景：**
用户让我分析 http://www.xyptdq.com 网站内容。

**错误做法：**
- 第一次依赖训练数据或搜索引擎信息
- 描述的内容与实际页面不符

**正确做法：**
1. 使用 `browser open` 打开目标 URL
2. 使用 `browser snapshot` 获取实际渲染内容
3. 基于实际获取的内容进行分析
4. 如果不确定，再次获取验证

**验证结果：**
实际页面内容是各类娱乐平台（门徒娱乐、欧陆娱乐、彩娱乐等）的推广页面，与之前描述完全不同。

**关键原则：**
> 永远基于实时获取的证据，不依赖记忆或间接信息。

---

## 进化机制

**每次遇到类似情况时：**
1. 先尝试 browser 工具获取实际内容
2. 如果失败，尝试 web_fetch 作为备选
3. 绝不凭空描述未验证的内容
4. 记录教训到 memory 文件
