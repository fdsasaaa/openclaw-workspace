## 三、安全层建设

### 3.1 敏感信息定义

已明确以下敏感信息类别：
- API密钥
- 访问令牌
- 密码/凭证
- 个人身份信息
- 系统路径
- 网络配置

### 3.2 存储规范

**正确方式**:
- 专用配置文件 (`~/.config/`)
- 环境变量
- OpenClaw auth系统

**禁止方式**:
- 明文写入记忆文件
- 硬编码在脚本中
- 写入Git仓库

### 3.3 操作分级

**需人工确认**（高风险）:
- 删除/覆盖操作
- 系统配置修改
- 网络暴露
- 凭证操作
- 外网敏感操作

**可直接执行**（低风险）:
- 只读检查
- 创建新文件
- 写入报告/日志
- 用户级Python包安装

## 四、记忆层建设

### 4.1 记忆文件分类

| 类别 | 位置 | 用途 |
|------|------|------|
| 每日记忆 | `memory/YYYY-MM-DD.md` | 当日操作记录 |
| 长期记忆 | `workspace/MEMORY.md` | 持久化偏好 |
| 任务检查点 | `memory/progress-*.json` | 长任务续跑 |
| Token用量 | `memory/usage-*.json` | API统计 |
| 安全事件 | `memory/security-incident-*.md` | 安全记录 |

### 4.2 每日记忆模板

已定义标准模板，包含：
- 今日概要
- 关键决策
- 执行记录
- 新增/修改文件
- 风险与问题
- 待办事项
- 反思与改进

### 4.3 检查点JSON结构

已定义标准结构，包含：
- taskId, taskName
- createdAt, updatedAt
- totalSteps, completedSteps
- steps数组（含status/checkpoint）
- metadata（model/tokens/dataFile）

## 五、备份层建设

### 5.1 备份策略

| 类别 | 频率 | 保留版本 |
|------|------|----------|
| 配置备份 | 修改前自动 | 10个 |
| 工作区备份 | 每周 | 4个 |
| 记忆备份 | 每日 | 30天 |
| 完整快照 | 每月 | 3个 |

### 5.2 目录职责

| 目录 | 用途 | 清理策略 |
|------|------|----------|
| `memory/` | 记忆、检查点 | 90天后归档 |
| `logs/` | 操作日志 | 30天后清理 |
| `reports/` | 阶段报告 | 长期保留 |
| `temp/` | 临时文件 | 7天后清理 |
| `configs/` | 配置文件 | 长期保留 |

### 5.3 灾难恢复预案

已定义3种场景：
1. OpenClaw配置损坏 → 从config备份恢复
2. 工作区丢失 → 从workspace备份恢复
3. 记忆文件丢失 → 从memory备份恢复或依赖MEMORY.md
